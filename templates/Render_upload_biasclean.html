What's New in v3.0

BiasClean v3.0 introduces Audit-First Mode with traffic-light governance (🟢 Green / 🟡 Yellow / 🔴 Red) to assess dataset safety before mitigation. New features include:

  • Pre-Mitigation Audit: Traffic-light system (🟢🟡🔴) evaluates deployment readiness
  • Three Operating Modes: Audit-First (recommended), Audit-Only, or Legacy
  • SVM Fairness Enforcement: Optional model-level fairness constraints
  • Enhanced Monitoring: Stage-wise tracking with confidence intervals
  • 7 Domain Frameworks: Justice, Healthcare, Finance, Hiring, Education, Business, Governance
  • Advanced Reporting: HTML reports, visualizations, and deployment decision support

SVM Fairness Enforcement

Integrity-Constrained

Maintains data integrity with minimal sampling (typically 101-103% retention). Weight-prioritized mitigation focuses on highest-impact features first.

Regulatory Compliance
BiasClean v3.0 aligns with UK Equality Act 2010, GDPR Article 22, and sector-specific guidelines (FCA, CQC, ICO). Audit-first mode supports evidence-based deployment decisions.
Technical Implementation
Protected attributes: Ethnicity, Gender, Age, DisabilityStatus, SocioeconomicStatus, Region, MigrationStatus. Analysis uses chi-square/Fisher exact tests (α=0.05), weighted bias scoring, and optional SVM fairness constraints.
Data Privacy & Security
All processing occurs server-side with secure file handling. Original data is never stored permanently. Results are temporarily cached and automatically deleted after download.
